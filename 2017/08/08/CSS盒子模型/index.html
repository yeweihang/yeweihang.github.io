盒子模型

对于盒子模型来说，盒子的组成由四部分：内容 边框 外边距 内边距

外边距

  • 综合写法：
    • margin: 100px;代表上右下左都是100px;
    • margin: 100px 50px;代表上下100px,左右50px;
    • margin: 100px 50px 30px;代表上100px,右50px,下30px,左50px;
    • margin: 100px 50px 30px 20px;代表上100px,右50px,下30px,左20px;
  • 按照方向分开设置：
    • margin-left: 100px;
    • margin-right: 50px;
    • margin-top: 30px;
    • margin-bottom:20px

内边距

内边距的设置和外边距一致。

边框

  • 综合设置：

    • border: 线框 线型 颜色

      四个边框设置的属性相同。设置的三个值没有先后顺序

  • 按属性去分别综合设置

    • border-width: 设置方式同外边距
    • border-style: solid dashed dotted double;
      • solid 实线
      • dashed 虚线
      • dotted 点线
      • double 双线
    • border-color: red blue black green;
  • 按照方向去分别综合设置
    • border-top: 10px solid red;
    • border-right: 5px dashed blue;
    • border-bottom: 3px botted green;
    • border-left: 2px double black;
  • 按照方向和属性去设置每个边框
    • border-top-width: 10px;
    • border-top-style: solid;
    • border-top-color: red;

背景图片的设置

背景图片的设置，我们需要记住以上这些属性
backfround-image: url(img/zhayan.png);
这句是添加背景图片。URL当中可以是图片的相对路径也可以是绝对路径。再有，背景图片添加进来的时候默认是重复的

background-repeat: no-repreat;
由于背景图片添加进来默认是重复的，我们需要根据需要设置背景是否重复。
它的值可以是repeat-x，代表水平方向可以重复。
还可以是repeat-y，代表垂直方向向下可以重复。
还可以是no-repreat，代表不重复。

background-size: 300px 200px;
background-size： 100% 100%；
它可以设置背景的大小，通常我们的图片都有默认大小，如果背景图片太大会超出盒子的范围，盒子范围之外的是看不到的，那么我们看到的背景不完整。如果背景图片太小，图片又不能完全覆盖盒子，盒子会流出空隙。因此，我们想让背景和盒子相同大小，需要给背景设置大小和盒子一样大小，可以用固定px去设置，也可以根据百分比去设置，百分比设置100% 100%；

background-position: -50px -20px;
这个属性可以定位我们的背景图片，让背景图片在盒子当中去任意移动。
如果图片比盒子小那么图片可以放置在盒子内的任意一个位置。比如图片放在何止中间我们给值 50% 50%; 放在右下角给值 100% 100%。放置在自己任意指定的位置，用固定的px值。
如果图片比盒子大，可以通过这个属性指定图片的哪部分显示在盒子当中，这就是我们说的雪碧图技术。

在css中需要注意的一些操作和问题

  • 文本水平居中

    所有行内标签都和文本一样

  • 单行文本在盒内正中

  • 多行文本在盒内居中

    多行文本没办法用单行文本的方式
    只能用盒子的padding去把文本往下挤
    但是这牵扯到盒子会变大的问题，我们需要精确还原盒子。

    • 精确还原盒子

      如果网站要求有一个固定大小的盒子，比如就是300乘以300的盒子，而我们制作网页的时候需要将盒子的内容往里挤我们添加了padding，这会盒子是会变大，不符合我们固定的大小。要去精确还原盒子精确还原盒子就是在哪个方向上有边距，就在哪个方向上减去相应大小的内容区域

  • 盒子居中
    margin: 0 auto;
    这个只能在标准文档流中用，且盒子的宽必须设置(浮动的元素将失效)

父子盒子垂直方向的margin转移
在处理父子盒子垂直方向边距时要善于运用父亲的padding，而不是儿子的margin，在处理兄弟的时候都可以。
标准文档流中兄弟盒子垂直方向的margin塌陷问题
兄弟盒子在垂直方向上，哥哥设置下外边距，弟弟设置上外边距。那么他们的外边距会有塌陷现象。小的边距会塌陷到大的边距里面。比如哥哥设置下外边距200，弟弟设置上外边距100，那么他们之间的外边距只有200，而不是300。
行内标签在垂直方向的边距不起作用，无论是内边距还是外边距
