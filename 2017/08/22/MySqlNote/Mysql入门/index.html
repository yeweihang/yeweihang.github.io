Mysql入门(一)

发表于 2017-08-22   |   分类于 Mysql   |   评论

SQL

结构化查询语言（Structured Query Language）

SQL语法

  • SQL语句可以自单行或多行书写，以分号结尾
  • 可以使用空格和缩进来增强语句的可读性
  • MySQL不区分大小写，建议使用大写

SQL语句分类

  • DDL（Date Definition Language）：数据定义语言，用来定义数据库对象：库、表、列等；创建、删除、修改、
  • DML（Data Manipulation Language）：数据操作语言，用来定义数据库记录；增删改查
  • DCL（Data Control Langage）：数据控制语言，用来定义访问权限和安全级别
  • DQL（Data Query Language）：数据查询语言，用来查询记录

DDL

1、数据库

  • 查看数据库：SHOW DATABASES;
  • 切换数据库：USE dbname;
  • 创建数据库：CREATE DATABASE [IF NOT EXITSTS] dbname [CHARSET=utf8];
  • 删除数据库：DROP DATABASE [IF EXISTS] dbname;
  • 修改数据库编码：ALTER DATABASE dbname CHARACTER SET uft8;

2、数据类型

  • int：整型
  • double：浮点型，例如double(5,2)表示最多5位，其中必须有2位小数，即最大值为999.99
  • decimal：浮点型，在表示钱方面使用该类型，因为不会出现精度缺失的问题
  • char：固定长度字符类型；char(255)，数据的长度不足指定长度，使用空格补足到指定长度
  • varchar：可变长度字符长度类型：varchar(65535)，需要一个字节来记录长度
  • text(clob)：字符串类型：
    • tinytext：很小，2的8次方减一
    • text：小，2的18次方减一
    • mediumtext：中，2的24次方减一
    • longtext：大，2的32次方减一，就是4G
  • blob：字节类型
    • tinyblob：很小，2的8次方减一
    • blob：小，2的18次方减一
    • mediumblob中，2的24次方减一,就是16M
    • longblob大，2的32次方减一，就是4G
  • date：日期类型，格式为：yyyy-MM-dd;
  • time：时间类型，格式为：hh:mm:ss
  • timestamp：时间戳类型

3、表

  • 创建表

  • 查看表：SHOW TABLES;

  • 查看表结构：DESC
  • 查看指定表的创建语句：SHOW CREATE TABLE tbname \G;
  • 删除表：DROP TABLE tbname;
  • 修改表：ALTER TABLE tbname
    • 添加列：ALTER TABLE tbname ADD (列名 列类型，…);
    • 修改列类型：ALTER TABLE tbname MODIFY 列名 列类型；
    • 修改列名：ALTER TABLE tbname CHANGE 原列名 新列名 列类型;
    • 删除列：ALTER TABLE tbanme DROP 列名;
    • 修改表名：ALTER TABLE tbname RENAME TO 新表名;

DML

  • 插入数据：INSERT INTO tbname(列名1,列名2,…) VALUES(列值1,列值2,…);

  • 修改数据：UPDATE 表名 SET 列名1=new列值1,列名2=new列值2,…[WHERE 条件];

    • 条件必须是一个boolean类型的值或表达式
    • 运算符：=、!=、<>、<、>、>=、<=、BETWEEN…AND…、IN(…)、IS NULL、NOT、OR、AND
  • 删除数据：
    • DELETE FROM 表名 [WHERE 条件];
    • TRUNCATE TABLE 表名;TRUNCATE是DDL语句，它是先删除drop该表，在create该表。而且无法回滚。

DCL

  • 创建用户
    • CREATE USER 用户名@IP地址 IDENTIFIED BY’密码’，创建一个用户只能在指定的IP地址登录
    • CREATE USER 用户名@’%’ IDENTIFIED BY’密码’，用户可以在任意的IP地址上登录
  • 给用户授权
    • GRANT 权限1,权限2,…,权限n ON 数据库.* TO 用户名@IP地址
    • GRANT CREATE,ALTER,DROP,INSERT,UPDATE,DELETE,SELECT ON mydb1.* TO Digreed@localhost;指定的权限
    • GRANT ALL ON mydb1.* TO Digreed@localhost;所有的权限
  • 撤销授权
    • REVOKE 权限1,权限2,…,权限n ON 数据库.* FROM 用户名@IP地址
    • REVOKE DELETE ON mydb1.* FROM Digreed@localhost;
  • 查看权限
    • SHOW GRANTS FOR 用户名@IP
    • SHOW GRANTS FOR Digreed@localhost;
  • 删除用户
    • DROP USER 用户名@IP地址

DQL

基本查询

  • 查询所有列：SELECT * FROM 表名;
  • 查询指定列：SELECT 列1,列2,…,列n FROM 表名;
  • 完全重复的记录只出一次：关键字DISTINCT
    • 当查询的结果中的多行记录一模一样时，只显示一次，一般查询所有列是很少会有这种情况，但只查询一列或几列时这种可能性就大了；
    • SELECT DISTINCT 列名 FROM 表名;
  • 列运算
    • 数量类型的列可以做加、减、乘、除运算，如果是其他类型，那么就会把它当成0来进行运算
    • SELECT sal*1.5 FROM emp;
    • 字符串类型可以做连续运算
    • SELECT CONCAT(‘$’,sal) FROM emp;
    • 转换NULL值：有时需要把NULL转换成其他值，例如comm+1000时，如果comm列存在NULL值，那么NULL+1000还是NULL，而我们这时希望把NULL当成0来运算。IFNULL(列名,NULL将转换成的值)
    • SELECT IFNULL(‘comm’,0)+1000 FROM emp;
    • 给列起别名：当使用列运算后，查询出的结果集中的列名称很不好看，这时我们需要给列起个别名，这样在结果中列名就显示别名，关键字AS，其实AS可以省略。
    • SELECT IFNULL(‘comm’,0)+1000 AS 奖金 FROM emp;

条件控制

  • 条件查询：与前面的UPDATA类似
  • 模糊查询：当你想查询姓张，并且姓名一共两个字的员工时，这是就可以使用模糊查询
    • SELECT * FROM emp WHERE ename LIKE ‘张‘;查出张某的员工，表示一个字符
    • SELECT * FROM emp WHERE ename LIKE ‘张%’;查出姓张的所有员工，%表示0~n个字符
    • SELECT * FROM emp WHERE ename LIKE ‘%小%’;查出名字带小字的员工
    • SELECT * FROM emp WHERE ename LIKE ‘%’;这个条件等同与不存在，但如果姓名为NULL的查询不出来！
  • 排序
    • 升序：SELECT * FROM WHERE emp ORDER BY sal ASC;
      • 这个ASC可以省略的
    • 降序：SELECT * FROM WHERE emp ORDER BY sal DESC;
      • 这个DESC不能省略！
    • 使用多列排序作为排序条件：
      • SELECT * FROM WHERE emp ORDER BY sal ASC,comm DESC;使用sal升序排，如果sal相同时，使用comm的降序排

聚合函数

  • COUNT
    • SELECT COUNT(*) FROM emp;计算emp表中所有列都不为NULL的记录的行数
    • SELECT COUNT(comm) FROM emp;计算emp表中comm列不为NULL的记录的行数
  • MAX
    • SELECT MAX(sal) AS 最高工资 FROM emp;查出最高工资
  • MIN
    • SELECT MIN(sal) AS 最低工资 FROM emp;查出最低工资
  • SUM
    • SELECT SUM(sal) AS 工资总和 FROM emp;查出工资总和
  • AVG
    • SELECT AVG(sal) AS 平均工资 FROM emp;查出平均工资

分组查询

分组查询是把记录使用某一列进行分组，然后查询组信息

  • SELECT deptno,COUNT(*) FROM emp GROUP BY deptno;使用deptno分组，查询部门编号和每个部门的记录数
  • SELECT job,MAX(sal) FROM emp GROUP BY job;使用job分组，查询每种工作的最高工资
  • 组条件，以部门分组，查询每组记录数。条件为记录数大于3
    SELECT deptno,COUNT(*) FROM emp GROUP BY deptno HAVING COUNT(*)>3;

LIMIT子句

LIMIT用来限定查询结果的起始行，以及总行数。

  • 查询起始行为第5行，一共查询3行记录：SELECT * FROM emp LIMIT 4,3;
  • 用于单页查询
    • 一页的记录数：10行;查询第3页：SELECT * FROM emp LIMIT 20,10;
    • (当前页-1)*每页记录数

编码问题

[client]
port=3306
[mysql]
default-character-set=utf8
改成
[client]
port=3306
[mysql]
default-character-set=gbk

#（修改了client、connection、results）

备份与恢复

  • 备份：mysqldump -u用户名 -p密码 数据库名>生成脚本路径
    • mysqldump -uroot -p888888 mydb1>c:/mydb1.sql
  • 恢复：mysql -u用户名 -p密码 数据库名<sql脚本路径
    • mysql -u用户名 -p密码 数据库名<c:/mydb1.sql
    • 先要创建数据库才可以恢复
  • 恢复：source c:/mydb1.sql;也是先要创建数据库
