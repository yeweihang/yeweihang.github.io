Mysql入门(二)

发表于 2017-08-22   |   分类于 Mysql   |   评论

约束

主键约束

  • 特点
    • 非空
    • 唯一
    • 被引用
  • 当表的某一列被指定为主键后，该列就不能为空，不能有重复值出现

  • 创建表示指定主键的两种方式

  • 修改表示指定主键：ALTER TABLE emp ADD PRIMARY KEY(empno);

  • 删除主键：ALTER TABLE emp DROP PRIMARY KEY;

    主键自增长

  • 因为主键列的特性是：必须唯一，不能为空，所以我们通常会指定主键类的类型为整型，然后设置其自动增长，这样可以保证子啊插入数据是主键列的唯一和非空特性

  • 创建表示指定主键自增长

  • 修改表时设置主键自增长：ALTER TABLE emp CHANGE empno empno INT AUTO_INCREMENT;

  • 修改表示删除主键自增长：ALTER TABLE emp CHANGE empno empno INT;

    非空约束和唯一约束

  • 非空约束：NOT NULL
  • 唯一约束：UNIQUE

概念模型

一对多
多对一
多对多
对象模型：可以双向关联，而且引用的是对象，而不是一个主键
关系模型：只能多方引用一方，而且引用的只是主键，而不是以整行记录

外键约束

  • 外键必须是另一个表的主键的值(外键要引用主键)
  • 外键可以重复
  • 外键可以为空
  • 一张表可以有多个外键

概念模型在数据库中成为表
数据库表中的多对一关系，只需要在多方使用一个独立的列来引用一方的主键

  • 创建外键的两种方式

多表查询

合并结果集

  • 要求被合并的表中，列的类型和列数相同
  • UNION,去除重复行
  • UNION ALL,不去除重复行

连接查询

  • 内连接
  • 外连接
    • 左外连接
    • 右外连接
    • 全外连接(MySQL不支持)
  • 自然连接

内连接

  • 方言：SELECT * FROM 表1,表2 WHERE 表1.xx=表2.xx
  • 标准：SELECT * FROM 表1 INNER JOIN 表2 ON 表1.xx=表2.xx
  • 自然：SELECT * FROM 表1 NATURAL JOIN 表2

外连接

  • 左外：SELECT * FROM 表1 LEFT OUTER JOIN 表2 ON 表1.xx=表2.xx
    • 左表记录无论是否满足条件都会查询出来，而右表只有满足条件才能出来。左表中不满足条件的记录，右表部分都为NULL
  • 左外自然：SELECT * FROM 表1 NATURAL LEFT OUTER JOIN 表2 ON 表1.xx=表2.xx
  • 右外：SELECT * FROM 表1 RIGHT OUTER JOIN 表2 ON 表1.xx=表2.xx
    • 右表记录无论是否满足条件都会查询出来，而左表只有满足条件才能出来。右表不满足条件的记录，其左表部分都为NULL
  • 右外自然：SELECT * FROM 表1 NATURAL RIGHT OUTER JOIN 表2 ON 表1.xx=表2.xx
  • 全连接：可以使用UNION来完成全连接

子查询

查询中有查询

  • 出现的位置
    • WHERE后作为条件存在
    • FROM后作为表存在
  • 条件
    • 单行单列：SELECT * FROM 表1 WHERE 列1 [=、>、<、>=、<=、!=] (SELECT 列 FROM 表2 WHERE 条件
    • 多行单列：SELECT * FROM 表1 WHERE 列1 [IN、ALL、ANY] (SELECT 列 FROM 表2 WHERE 条件)
    • 单行多列：SELECT FROM 表1 WHERE (列1,列2) IN (SELECT FROM 表2 WHERE 条件)
    • 多行多列：SELECT * FROM 表1,(SELECT …) WHERE 条件
