Java 概述

发表于 2017-05-21   |   分类于 JavaNote   |   评论

Java语言的发展历史

  • Java之父——James Gosling
  • Java1.0——1996.1

    发布了运行环境(JRE)和开发环境(JDK)。

  • Java1.4——2002.2.13

    是历史上最成熟的版本，主要是对JDK性能的改进，这时候JDK已经得到了广泛应用。

  • Java5.0——2004.9

    Sun将JDK改名为J2SE5.0,重点提高了语言的易用性。增加了泛型、增强for循环、可变数目参数、注释(Annotations)、自动拆箱和装箱等功能

  • 2009.4.20Sun公司被Oracle公司收购
  • Java7.0——2011.7

    提供了模块化支持，增强了对多种动态语言的支持。

Java7的架构

JDK、JRE和JVM的作用和关系

  • 作用

    JDK：Java的开发环境
    JRE：Java的运行环境
    JVM：Java跨平台特性的保证

  • 关系

    JDK：JRE+开发工具
    JRE：JVM+类库

Java语言的特点

  • 跨平台性(Java的最大特点)

    由Java虚拟机保证。

  • 面向对象

    支持类和对象、封装、继承、多态等面向对象的思想

  • 开源
  • 支持分布式
  • 支持多线程
  • 动态性
  • 结构中立
  • 安全
  • 。。。。。。

    配置环境

    1、安装Java的jdk

    2、设置环境变量

    1. 打开我的电脑–属性–高级–环境变量：
    2. 新建系统变量JAVA_HOME 和CLASSPATH
      变量名：JAVA_HOME
      变量值：D:\Advance\Java\jdk1.8.0_131
      变量名：CLASSPATH
      变量值：.;%JAVA_HOME%\lib\dt.jar;%JAVA_HOME%\lib\tools.jar;
      其中JAVA_HOME变量值为你自己安装jdk的文件目录，根据自己的情况修改。
      其中CLASSPATH变量值最前面的一个点不能丢。
    3. 选择“系统变量”中变量名为“Path”的环境变量，双击该变量，把JDK安装路径中bin目录的绝对路径，添加到Path变量的值中，并使用半角的分号和已有的路径进行分隔。
      变量名：Path
      变量值：%JAVA_HOME%\bin;%JAVA_HOME%\jre\bin;
      这是java的环境配置，配置完成后直接启动eclipse，它会自动完成java环境的配置。
    4. 测试环境变量的配置成功与否。在DOS命令行窗口输入“JAVAC”，输出帮助信息即为配置正确。如图：
